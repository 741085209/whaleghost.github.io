std::thread

线程构造器

构造器 说明
thread() noexcept; 默认构造函数
thread( thread&& other ) noexcept; 移动构造函数
template<class F, class… Args>
explicit thread( F&& f, Args&&… args );		 带参构造函数，f为可调用对象，args为可变参数列表
thread( const thread& ) = delete; 拷贝构造函数被禁用

构造参数

std::thread在调用template<class F, class... Args> explicit thread( F&& f, Args&&... args )时，args被捕获时，分为以下情况：

  • lvalue：args被拷贝构造
  • rvalue/xvalue：args被移动构造

无论何种情况都是传递副本，若有改变args值的需求，需为引用传递，需用std::ref或std::cref包装器进行包装。

构造细节

std::decay进行等价于按值传递时进行的类型转换。

传入的f与args会经过std::decay退化为原类型，在构造时会执行退化得到原类型并进行拷贝并移入std::thread中。

换言之，子线程捕获了构造时的f与args，并单独持有一份，至于执行何种构造/拷贝，需要看具体类型与拷贝构造函数实现。

成员函数

函数 说明
operator= 移动线程对象
joinable 检查线程是否可合并，即潜在运行于并行上下文之中
join 等待线程完成其执行
detach 容许线程从线程句柄独立开来执行
容易出现UB，尽量避免使用
get_id 返回线程的 id

detach下的危险行为

子线程执行detach后将挂到后台独立执行，被称为守护线程，已分配的资源由自身进行管理释放。

可调用对象或参数（class/struct）内含引用/指针

本质上是由于浅拷贝引起的悬垂引用，多个对象同时引用了同一块内存（非特指堆内存，也包括栈内存），只要其中一个对象析构，那么其余对象必将产生悬垂引用，从而产生未定义行为（UB）。可通过定义深拷贝构造函数解决。

使用全局对象

std::cout为全局对象，生命周期与主线程相同
